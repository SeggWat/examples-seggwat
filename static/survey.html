Survey Widget

Multi-question Surveys

Render targeted surveys as a slide-in banner, centered modal, or inline element. Triggers, copy, and questions are configured server-side — the widget just renders them.

Banner mode (default)

A fixed slide-in at the bottom of the viewport. Best for unobtrusive in-product research. Fires automatically when a server-side trigger (URL pattern, custom event, time-on-page) matches.

Embed Code

<script defer
  src="https://seggwat.com/static/widgets/v1/seggwat-survey.js"
  data-project-key="YOUR_PROJECT_KEY"
  data-button-color="#6366f1"
  data-mode="banner">
</script>

Programmatic control

"Fire custom event" dispatches seggwat:survey:demo — configure a survey in the dashboard with an event trigger named demo to react.

Modal mode

A centered overlay with a backdrop. Use for high-priority research moments like cancellation flows or post-onboarding interviews.

Embed Code

<script defer
  src="https://seggwat.com/static/widgets/v1/seggwat-survey.js"
  data-project-key="YOUR_PROJECT_KEY"
  data-mode="modal"
  data-survey-id="YOUR_SURVEY_ID">
</script>

Setting data-survey-id forces a specific survey and bypasses trigger evaluation. Cooldowns still apply unless you call SeggwatSurvey.show(id, { force: true }).

Inline mode

Mounts directly into a container you supply. No backdrop, no dismiss button — perfect for in-page research on docs, knowledge base, or empty-state screens.

<div id="inline-survey-container"></div>

<script defer

  data-mode="inline"

  data-survey-id="YOUR_SURVEY_ID">
</script>

Lifecycle invitations

Your backend dispatches lifecycle events (trial_expired, subscription_cancelled, custom) via the API, then injects the returned magic-link tokens here. The widget renders the matching survey in-app and suppresses the email follow-up.

Inject tokens from your backend

<!-- Option A: server renders tokens into the script tag -->
<script defer
  src="https://seggwat.com/static/widgets/v1/seggwat-survey.js"
  data-project-key="YOUR_PROJECT_KEY"
  data-pending-tokens="tok_abc123 tok_def456">
</script>

<!-- Option B: client-side after a route change -->
<script>
  const res = await fetch(
    `/api/v1/projects/${PROJECT_ID}/pending-invitations?user_ref=${userRef}`,
    { headers: { 'X-API-Key': ORG_TOKEN } }
  );
  const { tokens } = await res.json();
  SeggwatSurvey?.setPendingInvitations(tokens);
</script>

Configuration reference

Attribute Description Default
data-project-key Project UUID (required) —
data-mode Display mode: banner, modal, or inline banner
data-container CSS selector for inline mount point (inline mode) —
data-survey-id Force a specific survey, bypassing trigger evaluation —
data-cooldown-days Override the per-survey cooldown locally (dev/QA) Server value
data-pending-tokens Whitespace- or comma-separated magic-link tokens for lifecycle invitations —
data-button-color Hex color for the primary action button #2563eb
data-language UI language (en, de, sv, fr) Auto-detect
data-version App version stamped on submissions —
data-show-powered-by Show "Powered by SeggWat" footer true

Public API

// Identify the respondent (max 255 chars, alphanumeric + - _)
SeggwatSurvey.setUser('user_123');

// Fire a custom event → triggers any survey listening for 'checkout_completed'
SeggwatSurvey.trigger('checkout_completed');

// Show a specific survey now (respects cooldown unless forced)
SeggwatSurvey.show('srv_abc123');
SeggwatSurvey.show('srv_abc123', { force: true });

// Hide the active survey without submitting
SeggwatSurvey.hide();

// Inject lifecycle invitation tokens fetched from your backend
await SeggwatSurvey.setPendingInvitations(['tok_abc', 'tok_def']);

// Check whether the current user has answered within cooldown
if (!SeggwatSurvey.hasAnswered('srv_abc123')) { /* prompt them */ }

// Clear all local cooldown / sample-roll state (dev/QA)
SeggwatSurvey.reset();

// Read-only snapshot of live surveys returned by the server
console.log(SeggwatSurvey.surveys);
